# Excel Behavior Compatibility Matrix

This matrix ranks Excel behavior by release usefulness for real worksheets. It is not a
general validator backlog. A row belongs here only when it is backed by a real/public
workbook fixture, a package-action proof, or an explicit claim boundary that prevents
silent workbook corruption.

## Source Boundary

Microsoft documentation frames the Excel semantics used by the matrix: calcChain is
SpreadsheetML calculation-chain metadata, structured references and defined names are
first-class Excel formula surfaces, conditional formatting/data validation/autoFilter are
SpreadsheetML worksheet elements, and Protected View/macro/ActiveX settings are trust
and active-content workflows rather than parser correctness.

OSS baselines are intentionally treated as minimum bars, not parity targets. openpyxl
documents that unsupported workbook objects such as shapes may be lost on save, while
SheetJS CE documents writer scope and data-preservation limits. Ascend should surpass
that behavior for agent workflows by preserving/reporting unsupported package surfaces
instead of silently dropping them.

## Release Rule

Promote a row only when the open -> inspect -> edit safely -> save/reopen -> verify
ladder is green for real/public workbooks at that rank. If a workbook feature is not
fully supported, Ascend must preserve it unchanged, report it honestly, require an
explicit gate such as a password or owner approval, or fail closed.
