# Release Trust Matrix

This is the bounded correctness/trust matrix for the current release loop. It names the
failure classes Ascend is expected to catch or prove safe before an edit is trusted.
Do not broaden this matrix for isolated malformed fields unless the case can silently
corrupt a real workbook, invalidate an exact journal, or hide post-write drift.

## Stop Condition

Correctness/trust is release-ready for these classes when:

- the edit plan surfaces the risk before write, or the operation rejects before mutation;
- supported commits reopen and verify the same invariant;
- journal/verifier issues use stable machine-readable surfaces and reasons;
- package graph drift is either expected by operation semantics or remains unresolved and blocking.

## Release Discipline

This matrix is intentionally narrow. New validation work should be accepted only when it
adds a missing proof row above, fixes a failing proof, or blocks a realistic silent
corruption path. Cosmetic validation breadth, malformed-field enumeration, or new issue
vocabulary should wait until a release-blocking trust gap requires it.
