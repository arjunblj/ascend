# Generated Fixture Decision Evidence

## Question

Can the top product fixture blockers be turned into per-case owner decision evidence, instead of relying on broad prose about generated structural fixtures?

## Hypothesis

Yes. The release proof index already knows the current generated structural cases and the tracked fixture-scan results. It can derive a machine-readable owner handoff that lists each generated case, replacement evidence, decision needed, allowed use, and forbidden use while keeping the release gates missing.

## External sources checked

- GitHub repository limits: https://docs.github.com/en/repositories/creating-and-managing-repositories/repository-limits
- GitHub large-file guidance: https://docs.github.com/github/managing-large-files/working-with-large-files
- OpenSSF Scorecard binary-artifacts check: https://github.com/ossf/scorecard/blob/main/docs/checks.md
- SLSA provenance specification: https://slsa.dev/spec/v1.0-rc1/provenance
- GitHub artifact attestations: https://docs.github.com/actions/concepts/security/artifact-attestations

## Why this matters to Ascend

The North Star needs trustworthy release claims. Generated topology fixtures are useful for hard-to-find XLSX package edge cases, but they are easy to overstate. Product owners need a case-by-case decision board that separates structural proof from public binary evidence, licensing, privacy, provenance, trust, and safety claims.

## Probe/implementation

- Inspected `safe-open-fixture-scan.ts`, `package-action-fixture-scan.ts`, and `release-proof-index.ts`.
- Added `generatedFixtureDecisionEvidence` to `fixtures/benchmarks/release-proof-index.ts` and owner-handoff JSON.
- The evidence covers:
  - `safe-open-proof/signed`
  - `safe-open-proof/unknown-part`
  - `safe-open-proof/malformed`
  - `package-action-proof/signature-invalidation-drop`
  - `package-action-proof/unknown-part-error`
- Each case records tracked replacement evidence, owner decision needed, allowed use, and forbidden use.
- Added regression coverage in `fixtures/benchmarks/release-proof-index.test.ts`.

## Confidence

High for the local proof shape because the object is derived from existing fixture policy and scan evidence and is covered by tests. Medium for product approval readiness because owners still may require external public binary replacements or a different fixture acquisition policy.

## Fold-in decision

Promote to product and release loops as owner handoff evidence only. Do not mark `public-edge-fixtures` or `edge-fixture-policy` satisfied. Do not promote generated bytes as public binaries, signed provenance, or trust/safety evidence.
