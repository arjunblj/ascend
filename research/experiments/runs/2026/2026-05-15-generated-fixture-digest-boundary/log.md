# Generated Fixture Digest Boundary

## Question

Should generated edge-package proof cases carry deterministic byte digests in the release proof index?

## Hypothesis

Yes, but only for deterministic generated bytes. Generated workbooks can include variable package metadata, so they should stay disclosed by source kind and case name without being included in stable-shape digest expectations.

## External sources checked

- [GitHub artifact attestation provenance docs](https://docs.github.com/actions/security-for-github-actions/using-artifact-attestations/using-artifact-attestations-to-establish-provenance-for-builds) use `sha256:HEX_DIGEST` as the subject digest for provenance.
- [GitHub CLI attestation verification](https://cli.github.com/manual/gh_attestation_verify) describes attestations as claims about a subject artifact identified by digest.
- [SLSA provenance](https://slsa.dev/spec/v0.1/provenance) frames subjects and materials with digest metadata.

## Why this matters to Ascend

The release proof index now discloses generated versus public fixture provenance. Deterministic hashes for generated edge packages make that disclosure auditable without checking in large binary fixtures or implying signed provenance.

## Probe/implementation

- Added `inputSha256` to safe-open and package-action proof case results.
- Added `generatedCaseSha256` to release-index fixture provenance summaries.
- Included generated digests in Markdown as short displayed prefixes while retaining full hashes in JSON.
- Limited package-action digest publication to `generated-edge-package` cases. The probe initially included `generated-workbook` cases, but stable shape digests became nondeterministic because newly generated workbook packages can vary. Those cases remain disclosed by name/count only.
- Updated stable-shape hashing to strip `inputSha256` from proof result shape hashing; full release artifact JSON still carries the full digest evidence.

## Results

Validation passed:

```bash
bun test fixtures/benchmarks/release-proof-index.test.ts fixtures/benchmarks/package-action-proof.test.ts fixtures/benchmarks/safe-open-proof.test.ts
bunx biome check fixtures/benchmarks/release-proof-index.ts fixtures/benchmarks/release-proof-index.test.ts fixtures/benchmarks/package-action-proof.ts fixtures/benchmarks/package-action-proof.test.ts fixtures/benchmarks/safe-open-proof.ts fixtures/benchmarks/safe-open-proof.test.ts
bunx tsc --build
```

The failed intermediate probe was useful: including generated workbook SHA values in stable proof shape made the package-action stable digest differ across two no-timings runs. The final gate keeps deterministic generated edge-package digests while preserving stable shape determinism.

## Confidence

High for deterministic generated edge package disclosure. Medium for generated workbook handling; if those inputs must become release-proof artifacts, they need deterministic package metadata or public fixture replacements.

## Fold-in decision

Promote to release/product proof-index hygiene. Do not claim signed provenance or publish generated workbook digests as stable release evidence.

Allowed wording: "release proof artifacts now include full SHA-256 identifiers for deterministic generated edge-package cases while keeping generated workbook cases disclosed but not digest-promoted."
